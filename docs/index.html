mini-tcl

A small TCL interpreter written in portable Lua, distributable as a single self-contained native executable.

Try it wish (Tk) Install Build Reference GitHub

Try it in your browser

This runs the real mini-tcl.lua — the same interpreter that gets embedded into the native binary. It prefers the bundled PUC-Rio Lua compiled to WebAssembly (built with emscripten), and falls back to fengari (a Lua VM in JavaScript) when the WebAssembly module is unavailable.












Examples:
proc square {n} {return [expr {$n*$n}]} then
puts [square 9] ·
foreach i {a b c} {puts $i} ·
string toupper hello ·
Arrow keys browse input history.







Install


Download a prebuilt binary from the
releases page:
mini-tcl-static (Linux, fully static) or
mini-tcl.exe (Windows). No installation, no dependencies —
one executable containing the interpreter and the script.


./mini-tcl              # interactive REPL
./mini-tcl script.tcl a b   # run a script; $argv0/$argv/$argc are set







Build from source


git clone https://github.com/arkenidar/mini-tcl.lua
cd mini-tcl.lua
make            # downloads minilua.h once, compiles mini-tcl
make test       # run the smoke-test suite


How it works: a tiny bundled tool (bin2c) turns
mini-tcl.lua into a C byte array; main.c bundles the
complete PUC-Rio Lua interpreter via single-header
minilua.h, loads the embedded
script and passes command-line arguments through, exactly like the standalone
lua binary would. Links only against libc and libm.


Other targets: make static (fully static Linux binary),
make windows (MinGW cross-build), make install,
make dist. A CMake build is also provided:
cmake -B build && cmake --build build && ctest --test-dir build.


The interpreter is plain Lua (5.1–5.5 compatible), so it also runs directly:
lua mini-tcl.lua.

Syntax


Standard TCL word rules: {braces} verbatim,
"quotes" with $var, [command] and
backslash substitution; ; separates commands; # at
command position is a comment (so ;# gives trailing comments);
trailing \ continues a line.
